Act as an expert full-stack developer. Your task is to build a complete, serverless web application called "SlideBoss.ai". This application will allow users to generate a downloadable Microsoft PowerPoint (.pptx) presentation from a simple text description using the Google Gemini API. The entire application must be configured to be deployable on Vercel's free tier.

**Core Requirements:**
1.  **Functionality:** A user enters text into a textarea, clicks a "Generate" button, and after a loading state, receives a downloadable `.pptx` file generated by an AI.
2.  **Stateless & Serverless:** The application will have no database and no traditional always-on server. The backend logic will be a single Python serverless function.
3.  **Frontend:** A modern, responsive single-page application built with React and Tailwind CSS.
4.  **Backend:** A Python serverless function that securely handles the AI API call and presentation generation.
5.  **Deployment Target:** All configurations must be optimized for deployment on Vercel.

---

### **Part 1: Project Setup and Structure**

Create the following directory and file structure. This is critical for Vercel deployment.

### **Part 2: Frontend (React + Vite + Tailwind CSS)**

**Location:** `/frontend/`

1.  **Initialize:** Use Vite to set up a React + TypeScript project in the `frontend` directory.
2.  **Install Dependencies:** Add `tailwindcss`, `postcss`, `autoprefixer`, and `axios`. Also, add `file-saver` and its types `@types/file-saver`.
3.  **Configure Tailwind:** Create `tailwind.config.js` and `postcss.config.js` and configure them correctly. Update `index.css` with the required Tailwind directives.
4.  **Create the Main Component (`App.tsx`):**
    *   This will be the only component. It must manage the application's state: `prompt` (the user's text), `isLoading` (boolean), and `error` (string).
    *   **UI Layout:**
        *   A header with the name "**SlideBoss.ai**" and the motto "**From Idea to Impact, Instantly.**"
        *   A main content area with an `h1` heading: "**Free AI-Powered Presentation Generator**".
        *   A `textarea` for user input, linked to the `prompt` state.
        *   A "Generate Slides" button. This button should be disabled when `isLoading` is true.
        *   When `isLoading` is true, display a loading indicator (e.g., a spinner and the text "Generating your presentation, please wait...").
        *   If an `error` occurs, display the error message.
    *   **Functionality (`handleSubmit`):**
        *   When the "Generate Slides" button is clicked, trigger an async function `handleSubmit`.
        *   Set `isLoading` to true and clear any previous `error`.
        *   Use `axios` to make a `POST` request to our backend endpoint: `/api/generate`.
        *   The request body must be a JSON object: `{ "prompt": prompt }`.
        *   The `axios` request config must include `responseType: 'blob'` to handle the file response.
        *   On a successful response, use the `file-saver` library to `saveAs(response.data, "SlideBoss_presentation.pptx")`.
        *   In case of an error, set the `error` state with a user-friendly message.
        *   Finally, set `isLoading` back to false.

### **Part 3: Backend (Python Serverless Function)**

**Location:** `/api/`

1.  **Create `requirements.txt`:** Add the following dependencies:
    ```
    Flask
    python-pptx
    google-generativeai
    Flask-CORS
    ```

2.  **Create `generate.py`:**
    *   This file will contain a Flask application. Vercel automatically serves a file with a Flask `app` instance as a serverless function.
    *   Import necessary libraries: `os`, `io`, `Flask`, `request`, `send_file`, `CORS`, `pptx`, and `google.generativeai`.
    *   **Security:** Initialize the Gemini client by getting the API key from environment variables: `genai.configure(api_key=os.environ.get("GEMINI_API_KEY"))`. **DO NOT HARDCODE THE KEY.**
    *   **Flask App:** Create a Flask app instance (`app = Flask(__name__)`) and enable CORS (`CORS(app)`).
    *   **API Endpoint (`@app.route('/api/generate', methods=['POST'])`):**
        1.  Get the user's prompt from the request JSON: `data = request.get_json()`.
        2.  **Prompt Engineering:** Create a detailed prompt to send to the Gemini model. This prompt must instruct the AI to return a response ONLY in JSON format. The JSON should have a specific structure:
            ```json
            {
              "slides": [
                { "title": "Slide Title 1", "content": "Bulleted or paragraph content for slide 1." },
                { "title": "Slide Title 2", "content": "Content for slide 2." }
              ]
            }
            ```
            Your instruction to Gemini should be very clear about this format.
        3.  Call the Gemini API with your engineered prompt.
        4.  Parse the JSON response from Gemini.
        5.  **Presentation Generation (`python-pptx`):**
            *   Create an in-memory byte stream: `prs_buffer = io.BytesIO()`.
            *   Create a `pptx.Presentation()` object.
            *   Define a title slide layout and a content slide layout.
            *   Loop through the `slides` array from the Gemini JSON response.
            *   For each item, add a new slide to the presentation using the appropriate layout, and populate the title and content placeholders.
            *   Save the final presentation to the in-memory buffer: `prs.save(prs_buffer)`.
            *   Reset the buffer's position: `prs_buffer.seek(0)`.
        6.  **Return File:** Use Flask's `send_file` to return the presentation. Set the `mimetype` to `application/vnd.openxmlformats-officedocument.presentationml.presentation` and `as_attachment=True`, with `download_name='presentation.pptx'`.

### **Part 4: Vercel Deployment Configuration**

1.  **Create `vercel.json` at the root:** This file tells Vercel how to handle the monorepo structure.
    ```json
    {
      "version": 2,
      "builds": [
        {
          "src": "frontend/package.json",
          "use": "@vercel/static-build",
          "config": { "distDir": "frontend/dist" }
        },
        {
          "src": "api/generate.py",
          "use": "@vercel/python"
        }
      ],
      "routes": [
        {
          "src": "/api/(.*)",
          "dest": "api/generate.py"
        },
        {
          "src": "/(.*)",
          "dest": "frontend/dist/index.html"
        }
      ]
    }
    ```

2.  **Create `.gitignore` at the root:**
    ```
    # Python
    __pycache__/
    *.pyc
    .env
    venv/

    # Node
    frontend/node_modules/
    frontend/dist/
    frontend/.DS_Store
    ```

After generating the code based on this prompt, the user will only need to add their `GEMINI_API_KEY` as an environment variable in their Vercel project settings to deploy the live application.
