Act as an expert full-stack developer and SEO specialist. Your task is to build a complete, serverless web application called "SlideBoss.ai". This application will be ready to deploy on Vercel's free tier and will be optimized for search engines from the start.

### **Part 3: Frontend Implementation (React + Vite + Tailwind CSS)**

**Location:** `/frontend/`

1.  **Initialize & Install:** Use Vite for React + TypeScript. Install `tailwindcss`, `postcss`, `autoprefixer`, `axios`, and `file-saver` (with `@types/file-saver`).
2.  **Configure `index.html`:** Update the `frontend/index.html` file to include the exact Title and Meta Description tags from Part 1.
3.  **Configure Tailwind:** Set up `tailwind.config.js`, `postcss.config.js`, and `frontend/src/index.css`.
4.  **Build the Main Component (`App.tsx`):**
    *   Create a single component that holds the entire UI and application logic.
    *   **State Management:** Use `useState` for `prompt` (string), `isLoading` (boolean), and `error` (string).
    *   **UI Construction:**
        *   Build the UI using semantic HTML. Use the exact H1, H2, H3, and paragraph text specified in the **On-Page Content Strategy** from Part 1.
        *   Style the page cleanly using Tailwind CSS.
        *   The Call-to-Action section must contain the `textarea` and the "Generate Slides" button.
        *   Implement a loading state (spinner/message) that appears when `isLoading` is true.
        *   Display the `error` message if one occurs.
    *   **Client-Side Logic (`handleSubmit`):**
        *   On button click, set `isLoading` to true.
        *   Make a `POST` request to `/api/generate` using `axios`, sending `{ "prompt": prompt }` in the body. The request MUST have `responseType: 'blob'`.
        *   On success, use `file-saver` to save the blob as `SlideBoss_presentation.pptx`.
        *   On error, populate the `error` state.
        *   Set `isLoading` to false in all cases (success or error).

### **Part 4: Backend Implementation (Python Serverless Function)**

**Location:** `/api/`

1.  **Create `requirements.txt`:** Include `Flask`, `python-pptx`, `google-generativeai`, and `Flask-CORS`.
2.  **Create `generate.py`:**
    *   Set up a basic Flask `app`.
    *   **Security:** Configure the Gemini client using an environment variable: `os.environ.get("GEMINI_API_KEY")`.
    *   **API Endpoint (`/api/generate`):**
        *   This endpoint must accept `POST` requests.
        *   Get the user's prompt from the request body.
        *   **Crucial Prompt Engineering:** Create a system prompt for the Gemini model that commands it to respond ONLY with a JSON object structured exactly like this: `{"slides": [{"title": "Slide 1 Title", "content": "Slide 1 content as a string."}, ...]}`.
        *   Call the Gemini API.
        *   Parse the returned JSON.
        *   **PPTX Generation:** Use `python-pptx` to create a new presentation in memory (`io.BytesIO`).
        *   Iterate through the `slides` array from the AI's response. For each object, add a new slide to the presentation, populating the title and content placeholders.
        *   **Return the File:** Use Flask's `send_file` to return the in-memory presentation buffer as an attachment with the correct MIME type.

### **Part 5: Deployment Configuration (Vercel)**

**Location:** `/` (root)

1.  **Create `vercel.json`:** Use the following configuration to tell Vercel how to build the frontend and route requests to the backend function.

    ```json
    {
      "version": 2,
      "builds": [
        {
          "src": "frontend/package.json",
          "use": "@vercel/static-build",
          "config": { "distDir": "frontend/dist" }
        },
        {
          "src": "api/generate.py",
          "use": "@vercel/python"
        }
      ],
      "routes": [
        { "src": "/api/(.*)", "dest": "api/generate.py" },
        { "src": "/(.*)", "dest": "frontend/dist/index.html" }
      ]
    }
    ```
